# 活水来源设计 · 共识过滤 + 三级节奏

> 核心原则:**单一来源不可信,star 数可以刷,热搜可以炒。一项技术/仓库要进课堂,必须在"互不相识"的至少两类源里同时出现。**

## 一、来源矩阵(两类六源)

### A 类 · 权威/机器信号(脚本自动抓,无需登录)

| 源 | 信号含义 | 抓取方式 |
|---|---|---|
| GitHub Trending / star 增速 | 开发者用脚投票 | `gh search`(已在雷达脚本) |
| Hacker News 首页(≥100 分) | 全球工程师社区共识,刷不动 | Algolia 公开 API(已在脉搏脚本) |
| HuggingFace Trending | 模型/数据集侧的真实热度 | 公开 API(已在脉搏脚本) |
| 官方实验室博客(Anthropic/OpenAI/Google/阿里通义/DeepSeek) | 一手权威,可免冷静期 | NotebookLM「发现来源」或智能体联网 |

### B 类 · 社区/自媒体信号(智能体口令 + NotebookLM 抓)

| 源 | 用途 | 获取方式 |
|---|---|---|
| 机器之心 / 量子位 / 新智元(公众号) | 中文头部科技媒体,过滤后的行业共识 | 智能体口令"每日脉搏"联网检索;或 NotebookLM Discover |
| Reddit r/LocalLLaMA、r/MachineLearning 高赞 | 一线实践者真实反馈(某工具"实际不好用"往往先在这暴露) | 智能体联网检索 |
| B站/YouTube 头部技术 UP 主实测视频 | 学生喜闻乐见的验证渠道,适合布置预习 | 教师每周精选 1 条 |
| V2EX / 掘金 / 知乎高赞长文 | 国内工程师踩坑一手记录 | 智能体联网检索 |

## 二、共识过滤器(什么才配进课堂)

**入雷达**(进周报即可):A 类任意一源命中。

**入课堂**(当解剖对象/教学工具),必须同时满足:
1. **双源规则**:≥2 类独立来源命中(如 GitHub 增速 + HN 首页;或 HF Trending + 机器之心报道);
2. **可跑规则**:有可运行代码、License 允许教学使用、30 天内有维护、issue 区有人应答;
3. **冷静期规则**:非官方实验室发布的爆红项目,观察 7 天再进课堂(防刷星、防"发布即巅峰");
4. **可教规则**:核心链路 2 小时内能被学生读懂,否则只讲思想不解剖。

**入任务卡**(改课程大纲级别):连续 4 周留在雷达 + 出现在真实 JD 里。技术栈写进任务卡是最重的决定,每学期集中校准一次。

## 三、三级节奏

| 节奏 | 动作 | 工具 | 产出 |
|---|---|---|---|
| **每 1–3 天 · 脉搏** | 跑 `daily_pulse.sh`(A类机器信号)+ 对智能体说"每日脉搏"(B类中文热点) | 脚本 + 智能体联网 | 3–5 条快讯贴进 Obsidian 日记,课前 5 分钟"行业早报"环节 |
| **每周 · 雷达** | 跑 `weekly_radar.sh`,按共识过滤器选出本周解剖仓库 | 脚本 + 智能体 | 雷达周报 → NotebookLM 情报播客 |
| **每学期 · 校准** | 对照 16 周雷达存档 + 最新校招 JD,修订八关任务卡技术栈 | 教师 + 智能体 | 任务卡 vN+1,这是教研成果本身(可发教改论文) |

## 四、防腐条款(顶级教研工具的自我约束)

- 雷达周报永远**存档不删**:16 周后你拥有一份"本学期 AI 技术演化编年史",既是教研资产,也是下学期校准依据;
- 每条进课堂的技术,记录**引入理由和来源证据**(哪两源命中),学期末回看命中率——教师自己的"选题 eval";
- 智能体输出的任何热点,必须附来源链接,学生可溯源;无法溯源的信息不进课堂。
