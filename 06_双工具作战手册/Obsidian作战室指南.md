# Obsidian 作战室指南(2026 能力全开版)

> 定位:教师的**指挥部**(本库)+ 每个学生的**求职作战室**(模板见 `07_学生作战室模板`)。核心升级:Bases 数据看板 + MCP 接入,让库从"笔记堆"变成"能被 AI 操作的活系统"。

## 教师指挥部(本库)

**必开插件**:Bases(核心自带,2026)、Templater、Excalidraw、Spaced Repetition(可选)。

1. **课程推进看板(Bases)**:每个学生一条笔记,YAML 属性:`小组` `当前关卡` `关1..关8`(通过/重修/未到)`风险`。Bases 建表格视图按小组分组、按关卡筛选——全班进度一屏尽收
2. **能力雷达**:学生属性含八关得分,期末导出成"能力矩阵"截图,家长会/督导材料直接用
3. **Canvas 课程地图**:八关依赖关系一张图,第一课展示
4. **Templater**:关卡任务卡、周复盘、面试战报全部做成模板,一键新建

## 学生作战室(第 1 关领取模板,详见 07 目录)

```
我的作战室/
├── 00_控制台.md          ← Bases 看板首页:关卡进度/待办/面试日程
├── 赛道/                 ← 第1关:JD库(Bases表)+ 赛道声明
├── 项目/                 ← 每关一个项目文档(架构图/决策记录)
├── 评测/                 ← 评测历史表(Bases:版本/通过率/成本)
├── 复盘/                 ← 每关一篇 + #failure-mode 失败笔记
└── 面试/                 ← 错题库/战报/简历版本
```

**纪律**:属性(YAML)是看板的燃料——每篇笔记必须填 `关卡` `状态` `标签`;链接代替文件夹嵌套;每周五 10 分钟"作战室保养"。

## 王牌用法:作战室 × MCP(第 5 关)

学生把自己的作战室做成 MCP server(社区已有 obsidian-claude-code-mcp、MCP Tools for Obsidian 等成熟参考实现,允许参考但要求自己实现核心工具):

- `search_notes` —— AI 检索你的复盘和错题
- `add_interview_question` —— 面试后一句话让 AI 把错题归档
- `get_progress` —— AI 读你的关卡状态,主动建议下一步

效果:**第 8 关面试冲刺时,学生对着 Claude 说"帮我复习我在 RAG 项目里踩过的坑",AI 直接检索他自己的失败笔记出题**——这个 Demo 本身就是面试爆点,也是"学了就用在自己身上"的完美闭环。

## 与 NotebookLM 的分工

| | Obsidian | NotebookLM |
|---|---|---|
| 角色 | 写(沉淀/结构化/看板) | 读(预习/测验/陪练) |
| 数据流 | 学生写复盘评测 → | ← 导出上传,生成播客/测验/批评 |
| 一句话 | 你的第二大脑 | 你的私人教研室 |
