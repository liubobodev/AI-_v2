## 四位一体怎么配合（教师视角）

```
GitHub（活水源） ──每周雷达/解剖仓库/学生作品集── 见 09_GitHub资源引擎
        │ gh CLI / GitHub MCP
        ▼
Obsidian（本库）          NotebookLM                 教练智能体
── 课程体系/任务卡/看板   ── 学生预习+测验+面试陪练     ── 课堂执行+项目诊断+模拟面试
   教师的指挥部              学生的学习引擎               每个学生的私人教练
        │                       ▲                          ▲
        └── 08_NotebookLM导入包 ─┘        01_智能体设定 ────┘
```

**每周固定循环**:周一跑 `09_GitHub资源引擎/weekly_radar.sh` 出雷达周报 → 上传 NotebookLM 生成本周情报播客 + 预习测验 → **课上第一段:教师高阶讲解（`12_高阶知识讲解`对应关卡文件，20–30分钟）** → 课上第二段:解剖本周仓库 + 推进关卡实战 → 学生 commit 进 GitHub、笔记进作战室 → 周末 Debate 模式备答辩。

**每关的四步节奏（写进每张任务卡的执行顺序）**:
```
自学基础（NotebookLM预习播客） → 教师高阶讲解（12_高阶知识讲解） → 实战（任务卡课堂任务） → 评测（智能体质询+四件套验收）
```

**开学前 30 分钟部署:**
1. 用 Obsidian 打开本文件夹（建议装:Bases[核心自带]、Templater、Excalidraw、Spaced Repetition）
2. 把 `08_NotebookLM导入包` 的文件上传到一个 NotebookLM 笔记本，开共享链接发给全班
3. 把 `01_智能体设定/实战总教练_系统提示词.md` 复制到你选的智能体平台（Claude/豆包/元宝/Coze 均可）
4. 学生第 1 周领取 `07_学生作战室模板`，建立自己的求职作战室
5. 启动 webapp 对话界面: `cd webapp && npm install && npm run dev`

## Web 界面

> 💡 **一键启动**：双击桌面上的「启动AI教练.command」，自动打开终端 + 浏览器。首次使用前先运行 `cd webapp && npm install`。


`webapp/` 是训练营的对话界面——不是静态 Prompt，是一个真的能对话、能调用 DeepSeek API 的 Next.js 应用。

**特性:**
- 左侧「八关进度」点击即可切换场景，自动加载该关专属知识库（按需加载，token 消耗降低 60-70%）
- 8 个快捷口令:讲高阶、记录卡点、选模块、选企业场景、每日脉搏、面试我、难题抽问、诊断项目
- 聊天记录 + 关卡进度自动保存到浏览器 localStorage，刷新不丢失
- 首次访问自动弹出 3 步引导:选身份 → 选关卡 → 开始训练
- API Key 只存浏览器 localStorage，不落盘

**本地运行:**
```bash
cd webapp
npm install
npm run dev
```
打开 http://localhost:3000，右上角「API Key(DeepSeek)」可切换为界面覆盖模式。

**知识库更新**:修改 `12_高阶知识讲解`、任务卡等文件后，重启 `npm run dev` 即可生效。

## v3「闯关驾驶舱」改造（本次新增）

针对"侧边栏各关都一样、回复像文章、不知道怎么算通关"的反馈，做了结构性翻转：

- **闯关地图**：登录后落地页不再是聊天框，是 8 张关卡卡片，每张显示三道门（自检/评审/面试）的点亮状态
- **关卡作战室**：点进任意关卡，三个标签页——「任务简报」（目标/JD对标/任务/简历句）、「高阶知识」（教师高阶讲解 + `13_AI应用开发能力体系` 补充，按知识点分卡片）、「验收自检」（勾选清单 + 提交 GitHub 链接真实评审 + 面试追问预演）
- **真实评审闭环**：提交链接后服务端真的抓取 GitHub 文件树/README/COST.md/POSTMORTEM.md（不是让模型假装读链接），组装证据包交给模型按本关 rubric 逐条打分，≥70 分且 criteria 通过才点亮「评审」门
- **三门通关判定**：自检 + 评审 + 面试全绿才算通关，写死在 `webapp/lib/studentStore.ts` 的状态机里，不再靠感觉
- **单一事实源**：`webapp/scripts/build-gates.ts` 从 Obsidian 库的任务卡 + 高阶讲解 + 能力体系文件自动解析出结构化关卡数据，改 md 文件重跑脚本即同步，不手写
- **聊天收进抽屉**：不再霸占主视图，浮动「💬 问教练」按钮弹出侧滑面板，仍保留全部原有功能（流式、多模型、快捷口令）
- **去机械化**：删除了客户端强制六段式格式检查，系统提示词改为"先结论、可枚举用表格、长度匹配问题重量"，不再每条回复都套用同一个模板
