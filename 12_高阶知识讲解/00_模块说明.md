# 高阶知识讲解模块 · 说明

> 这是 v2 相比 v1 最关键的一次升级:**每关的教学节奏从"自学→实战→评测"改为"自学→教师高阶讲解→实战→评测"。**
> 中间插入的这一段,就是教师水平的体现——学生自学能查到"是什么、怎么跑通",查不到"业界现在怎么做才算好、坑在哪、2026 年的共识是什么"。这段内容的深度,直接决定学生实战产出物能封顶到什么水平。

## 为什么必须单独成模块,而不是塞进任务卡

1. **需要持续更新**:高阶知识是会过期的(比如 2024 年教"朴素 RAG",2026 年企业已经在用混合检索+重排+agentic RAG 分流)。单独成模块,才能像 `09_GitHub资源引擎` 一样,靠"双源共识过滤"定期校准,不至于绑死在任务卡里逐年腐烂。
2. **需要分层**:本科生讲到"能落地的最佳实践"即可;硕士生要往上加一层"这些实践背后的原理、论文出处、可研究的开放问题"。每份文件都标了两档。
3. **教师是策展人,不是复述者**:每份文件末尾都附了可验证的来源链接和"深挖资源",教师课堂上不必照本宣科,而是挑 3–5 个点结合本届学生的场景现场展开——这才是"高阶"的意思:不是知识量大,是能把最新共识翻译成学生听得懂、马上能用的话。

## 使用方式(嵌入每周教学节奏)

```
周二:学生自学基础(NotebookLM 预习播客,对应任务卡"技术栈"一节)
课上第一段(20–30分钟):教师高阶讲解 ← 本模块对应关卡文件
    → 挑 3–5 个"本关实战钩子"里点名的技术点,现场演示或画图
    → 抛出"这一届你们要在实战里体现出这一点"的具体要求
课上第二段:学生实战(任务卡"课堂任务")
课后:评测 + 智能体质询(会按高阶知识点抽问,见各文件"面试延伸")
```

## 校准机制(防止过期)

- 学期初:教师用 WebSearch/gh 按每份文件的"来源锚点"关键词重新检索一轮,对照《09/活水来源设计》的双源规则,决定哪些点该替换;
- 学期末:把当届学生在这些高阶点上问出的好问题、发现的新资料,补进文件——本模块和 `09_GitHub资源引擎`、`11_学习难点飞轮` 一样,是会自己生长的活文档;
- 每份文件顶部标注"最近校准日期",超过 2 个学期未校准的文件,开课前必须优先重审。
