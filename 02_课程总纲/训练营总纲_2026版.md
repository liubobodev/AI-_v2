# AI 上岗实战训练营 · 总纲(2026 版)

## 课程定位

面向本科生与硕士生的**上岗型训练营**:不从理论教起(基础由 NotebookLM 预习引擎承担),课堂只做三件事——**做项目、被质询、攒证据**。

> 一句话目标:结营时,每个学生有一个**能演示、有评测、有成本记录、有复盘**的 GitHub 作品集,和一套**经过模拟面试压力测试**的表达。

## 为什么是这八关(对标 2026 招聘市场)

2026 年 AI 应用岗的筛选逻辑已经变了:
- 面试第一题不再是"你会调 API 吗",而是"**讲一个你设计过的评测**"
- JD 标配:MCP 集成、Agent 编排、RAG 工程化(切片/召回/重排)、Prompt 版本管理、可观测性、内容安全
- 国内校招 JD 直接写明:"深度使用 Claude Code、Cursor 等 AI 编程工具""设计可复用 Skill""搭建多 Agent 协作系统"
- 简历筛选标准:GitHub 上"干净 repo + eval traces + 成本看板 + 失败复盘" >> 花哨 Demo;开源参与/高星项目可直通二面

## 三舱(每周循环)

| 舱 | 干什么 | 载体 |
|---|---|---|
| 实战舱 | 每周做出可运行的东西 | 课堂 + Claude Code/Cursor |
| 面试舱 | 教练智能体按 4 种面试官人格质询 | 教练智能体 + NotebookLM Debate 模式 |
| 作品舱 | 沉淀简历句、GitHub 证据、复盘 | 学生 Obsidian 作战室 |

## 八关与 16 周执行表

| 周 | 关卡 | 核心产出 | 就业证据 |
|---|---|---|---|
| 1–2 | ① 侦察关 | 3 份产品拆解 + 10 份 JD 拆解 + 个人赛道声明 | 目标岗位画像 |
| 3–4 | ② AI 编程协作关 | spec→代码→测试 全链路小项目 | "熟练使用 AI 编程智能体进行规格驱动开发" |
| 5–6 | ③ Prompt 资产关 | 结构化输出小产品 + 模板版本库 + 20 条评测 | "Prompt 模板化与版本管理" |
| 7–8 | ④ RAG 工程关 | 带引用问答系统 + 召回率测试集 | "混合检索+重排,召回率 X%→Y%" |
| 9–10 | ⑤ 工具与 MCP 关 | 自建 MCP server(接 Obsidian 库) | "实现过 MCP server,接入真实系统" |
| 11–12 | ⑥ Agent 系统关 | 可规划/执行/恢复的 Agent(硕士:多 Agent) | "Agent 编排、记忆与失败处理" |
| 13–14 | ⑦ Evals 上线关 | 评测集 50+ 条、LLM judge、回归脚本、成本表、红队报告 | "评测集与自动化回归、A/B、可观测" |
| 15–16 | ⑧ 发射关 | GitHub 作品集 + 简历 + 路演 + 模拟终面 | 完整应聘弹药库 |

**通关规则**:每关期末答辩"三问"——为什么这么设计?试过什么别的方案?下一步最大风险?讲不清 = 该关重修。第 7 关是硬闸门:**没有评测的项目不允许进入发射关**。

## 本硕分层

| | 本科 | 硕士 |
|---|---|---|
| 通关标准 | 六关必过(②–⑦),①⑧全员 | 八关全过,⑥必须多 Agent |
| 深度加码 | — | 每关加:基线对比实验 + 半页技术报告;⑦关须含 LLM judge 与人工标注一致性分析 |
| 成绩构成 | 关卡通过 60% + 发射关 30% + 作战室质量 10% | 关卡 50% + 发射关 25% + 技术报告 25% |

## 课堂组织

教室 = AI 应用创业公司:3–4 人一组,组内分岗(工程/评测/产品),每周一次 15 分钟站会 + 一次交付评审。教师是 CTO,教练智能体是每人的 mentor,NotebookLM 是培训部,Obsidian 是公司 wiki。
