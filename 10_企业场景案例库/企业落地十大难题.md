# 企业落地十大难题 · 通用场景的真难点就是真教材

> 玩具 demo 和企业落地之间隔着这十道坎。**每道坎 = 一个教学模块 + 一道必考面试题。**
> 学生答辩时,教练智能体会从学生场景对应的难题里抽问(对照《场景选题矩阵》的"核心难点"列)。

| # | 难题 | 为什么难(真实世界) | 课堂怎么教(可操作) | 面试怎么答(得分点) |
|---|---|---|---|---|
| ① | **脏数据** | 企业数据 = 歪扫描、手写体、三行合并表头、口语转写稿;"数据清洗占落地工作量 60%" | 解剖关卡样例的清洗管道;作业强制用脏数据集,禁止用干净样例 | 讲清洗策略与放弃策略("哪些数据我判定不值得救") |
| ② | **知识冲突与过期** | 新旧版手册并存,答案取决于"哪份文档说了算" | RAG 关必做:元数据打版本标签、检索时过滤、冲突时显式提示用户 | "我按文档生效日期加权,冲突时同时展示两版并标注" |
| ③ | **幻觉零容忍场景** | 财务数字、合同条款、行动项归属——错一个=事故,90 分和 99 分是两个世界 | 教"生成+校验"双链路:LLM 提取 → 规则/二次模型校验 → 置信度分级 | "我不让模型裸奔:关键字段有校验器,过不了就转人工" |
| ④ | **偏见与合规红线** | 简历筛选涉就业歧视、数据涉个保法,企业比你想象的怕 | 红队作业:构造性别/年龄探针简历测自己的系统;学《生成式AI服务管理暂行办法》要点 | 能主动说出红线并展示探针测试结果 |
| ⑤ | **可解释性** | 业务方问"为什么给这个结论",答不出=不敢用 | 硬性要求:所有输出带引用原文/页码/口径说明 | "每个结论可以点开看依据,业务方可以否决" |
| ⑥ | **教会 AI 说"不知道"** | 拒答做不好两头挨骂:该答不答惹人烦,不该答乱答出事故 | 测试集里 20% 是"无法回答题",拒答率纳入评分 | 展示拒答策略的评测数据(误拒率/漏拒率) |
| ⑦ | **时效与来源可信** | 竞品情报、政策法规,过期信息比没有信息更危险 | 教来源分级(官方>媒体>论坛)+ 抓取时间戳必须入库 | "我的周报每条都有来源链接和抓取时间" |
| ⑧ | **无标准答案怎么评测** | 文案风格、纪要质量没有 ground truth,"感觉不错"不是验收 | Evals 关核心:把"感觉"拆成 rubric(如纪要=完整性/归属准确/无捏造 三维打分)+ LLM judge 对齐人工抽检 | 能现场写出一个 rubric 并解释 judge 与人工的一致率 |
| ⑨ | **人机协作断点设计** | 全自动是妄想,问题是"哪里断、怎么断":置信度阈值、审批点、回滚 | Agent 关必做:设计转人工的触发条件和交接摘要格式 | "我的系统知道自己什么时候不行,并且交接得干净" |
| ⑩ | **权限与隐私** | 报价是商密、工资是雷区;RAG 把全公司文档喂进去=事故预定 | MCP 关必做:工具层做权限过滤(谁问的→能看什么),敏感字段脱敏后才进上下文 | "检索层按提问者权限过滤,模型永远看不到无权数据" |

## 使用方式

1. **选题时**:学生定场景后,先到本表认领自己的 2–3 个难题,写进项目 README 的"本项目要攻克的真问题";
2. **教学时**:每关的"难点研讨"环节 = 本表对应行的展开(15 分钟案例 + 学生自己系统的对照检查);
3. **答辩时**:教练智能体按学生认领的难题抽问,答不上 = 该关"边缘通过",补交攻克方案;
4. **求职时**:每攻克一道难题 = 简历一句话 + 面试一个故事。十道题就是十个"我和培训班学员不一样"的证据。

> 教研提示:这张表本身每学期要校准——把当季企业咨询/学生实习带回的新难题补进来,这是这套工具"顶级"的另一半来源。
