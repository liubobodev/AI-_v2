# 基础准入 · 用 OpenMAIC 打地基,过准入自测再进八关

> 定位:本训练营是**上岗实战**训练营,不重复教基础理论——基础用清华开源的 **OpenMAIC** 自学,
> 回来做一次**轻量准入自测**,过了再解锁八关实战。这不是一道重考核关,是"进实战前的地基检查"。

## 为什么用 OpenMAIC 而不是自己讲基础

- OpenMAIC(THU-MAIC,清华开源多智能体互动课堂)专门优化"把知识讲懂":输入一个主题或 PDF,自动生成课件+语音+AI 老师讲解+AI 同学讨论+随堂测验。**它在"理解层"比我们强,免费,清华背书。**
- 我们守"认证层"(评审真作品、模拟面试、上岗)。两者是分层协作,不是竞争:**OpenMAIC 让你听懂,本系统让你被录用。**
- 项目地址:`THU-MAIC/OpenMAIC`(整合/外链前请自行核实许可证与最新仓库路径)。

## 一、基础自学清单(在 OpenMAIC 上逐个学懂)

进八关前,用 OpenMAIC 把这些概念学到"能用自己的话讲清":

| # | 概念 | 学到什么程度算够 |
|---|---|---|
| 1 | 大模型是怎么工作的(token/预测下一个词) | 能解释为什么模型会"胡编"、为什么有上下文长度限制 |
| 2 | Prompt 与结构化输出 | 知道角色/任务/约束/示例四要素,知道什么是 JSON schema 约束 |
| 3 | Embedding 与向量检索 | 能说清"语义相似=向量相近",知道 RAG 为什么需要它 |
| 4 | RAG 基本原理 | 能画出"切片→向量化→检索→拼进上下文→生成"这条链 |
| 5 | Agent 与工具调用 | 能区分"聊天机器人"和"会调工具、能多步执行的 Agent" |
| 6 | MCP 是什么、解决什么 | 一句话说清:让 Agent 用统一协议接外部工具/数据 |
| 7 | 什么是 Evals、为什么重要 | 知道"没有评测的 demo 不算产品",能说出至少两个评测指标 |

> 学法建议:每个概念在 OpenMAIC 生成一节互动课,听完 AI 老师讲 + 看 AI 同学提问,再做它的随堂测验。学完回本系统做准入自测。

## 二、准入自测题库(本系统出题,教练判定)

10 题,覆盖上面 7 个概念。**判定标准:答对/答得清 ≥7 题算通过,解锁八关。** 未过不惩罚,教练指出薄弱概念,回 OpenMAIC 补学再测。

1. 用一句话解释:大模型为什么会"一本正经地胡说八道"(幻觉)?
2. 上下文窗口是什么?它满了会发生什么?
3. 一个好的 Prompt 通常包含哪四个要素?
4. "让模型输出 JSON" 和 "用 schema 约束输出" 有什么区别?
5. Embedding 是什么?为什么说"语义相似就是向量相近"?
6. 画/描述一下 RAG 的完整链路(至少 4 步)。
7. Agent 和普通聊天机器人最大的区别是什么?
8. MCP 用一句话说清解决了什么问题?
9. 为什么说"没有评测(Evals)的 AI 项目只是 demo,不是产品"?
10. 举一个你能想到的 AI 应用场景,并说它可能会在哪一步"翻车"。

## 三、准入判定的教练话术(供智能体使用)

学生触发"基础自测"时,教练按上面 10 题逐题追问,基于回答判定:
- **通过(≥7 清晰)**:祝贺,点亮"地基已筑"徽章,引导进第 1 关侦察关。
- **边缘/未过**:指名薄弱的 2-3 个概念,给出对应的 OpenMAIC 自学建议,鼓励补学重测。
- 判定要基于学生**真的说清了**,不是背了名词。答"就是那个向量的东西"不算懂。

## 四、教师用法

- 开学第 1 周:布置学生用 OpenMAIC 学完自学清单,课上或课后完成准入自测。
- 准入自测成绩进学生档案,教师可在工作台看到"谁地基不牢",针对性补课。
- 这一层把"基础参差不齐"挡在实战八关之外,保证课堂时间真的花在实战上。
