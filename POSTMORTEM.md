# POSTMORTEM · 踩坑复盘(真实事故记录)

## #1 "六段式固定结构"让教练变八股(改了反而变差)

- **改动**:为治"回答散乱",在系统提示词强制六段式(判断→最小动作→步骤→验收→就业转化→下一步),前端还加了缺段检测追加警告;
- **结果**:学生反馈"回答机械像术语背诵",简单问题也被硬套六段,警告条反而打脸教练;
- **教训**:格式约束应按意图分级(诊断类结构化,闲聊类自由),而不是全局硬套;
- **修复**:UI 层弱化警告;后续版本在提示词里改为"可枚举信息用表格,简单问题直接答"。

## #2 vercel env pull 覆盖 .env.local(生产级事故)

- **现象**:学生问教练报"缺少 API Key"——部署操作把本地密钥文件整个冲掉,应用启动时无 env 校验,直到用户触发才炸;
- **教训**:必需 env 要有启动自检与显式报错;本地密钥与部署密钥分文件管理;
- **修复**:密钥已恢复并加注释警示;启动 env 自检列入待办。

## #3 默认模型选了思考型(glm-4.7-flash),学生盯空白 60 秒

- **现象**:思考流(reasoning_content)不被前端解析,首字延迟 30–60s,气泡一直"…";
- **教训**:教学场景默认模型应选低延迟非思考型;思考型作为可选升级;思考流应可视化("正在推理…");
- **修复**:对照测试确认 glm-4-flash 20s 内出完整回复;默认模型调整列入待办。
