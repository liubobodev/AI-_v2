# 学习难点飞轮 · 让教学内容自己生长

> 用户要的"融会贯通、不断生长、积累常见学习问题":靠制度不靠自觉。
> 核心机制:**每一次学生卡壳,都被强制转化为一张结构化"卡点卡片",汇入 Obsidian;
> 高频卡点自动浮出,变成下周的教学内容和 NotebookLM 测验题。教得越久,教材越厚,下一届起点越高。**

## 一、25 分钟规则(触发点)

学生卡住超过 25 分钟 → 必须问教练智能体(口令"记录卡点")→ 解决后 3 分钟内填卡。
不许硬扛(浪费课时),也不许秒问(丧失训练)。

## 二、卡点卡片模板(存入学生作战室 `卡点/` 目录)

```markdown
---
类型: 卡点
关卡: 4        # 1-8
模块: B        # A-F,见工具生态总图
场景: 客服助手  # 场景选题矩阵编号或名称
难题标签: 难题②知识冲突   # 对应企业落地十大难题,可空
耗时分钟: 45
解决方式: 智能体提示   # 自查文档/智能体提示/同学/教师/未解决
日期: 2026-10-13
---
# 卡点:切片后检索命中的全是目录页

**现象**(报错/异常输出原样贴):…
**卡住的原因**(解决后回头写,一句话):没过滤短文本块,目录页字少反而相似度高
**解法**(可复制的最小步骤):切片后加长度过滤 len>50,重建索引
**防再犯**(下次怎么提前发现):检索调试时先肉眼看 top5 命中内容再看分数
```

> 关键设计:YAML 属性齐全,卡片才能被 Bases 统计;`防再犯`一栏是训练元认知,面试讲踩坑故事的素材就从这来。

## 三、Bases 高频卡点榜(教师端,自动聚合)

在教师库建一个 Base,数据源指向全班作战室的 `卡点/` 目录(收作业时同步),视图:

| 视图 | 配置 | 用途 |
|---|---|---|
| 本周榜 | 按`日期`过滤本周,按`关卡`分组计数 | 周日备课:出现 ≥3 次的卡点 = 下周开场 15 分钟精讲 |
| 学期热力 | 按`关卡`ד难题标签`交叉 | 学期末看:哪一关哪类难题最费学生——任务卡该改哪 |
| 个人画像 | 按学生分组,看`解决方式`分布 | 谁总靠别人解决 → 单独辅导;谁全自查 → 树标杆 |

## 四、飞轮闭环(每周固定动作,并入 README 教学节奏)

```
学生卡壳 → 卡点卡片(Obsidian)
    → 周日:Bases 本周榜 Top3
        → ① 下周课堂开场"卡点复盘 15 分钟"(讲共性,不点名)
        → ② 喂给 NotebookLM:"根据这 3 个卡点生成 10 题测验" → 全班自测
        → ③ 教练智能体系统提示词追加一行"本届高频卡点提示"(学生再问到,秒给成熟解法)
    → 学期末:全部卡片 + 解法蒸馏成《本届卡点年鉴.md》
        → 下一届:任务卡预埋提示 + 智能体预载年鉴 = 教材完成一次进化
```

## 五、就业转化(卡点库的隐藏价值)

- 面试必考"你遇到过最难的问题?怎么解决的?"——学生的卡点库就是**带时间戳的真实踩坑履历**,比编故事真实一个量级;
- 第 8 关发射关的固定作业:从自己卡点库挑 3 张卡,改写成 STAR 格式面试故事;
- 硕士生:全班卡点数据本身可做教学研究(哪类难点与什么前置知识缺失相关)——又一个教改论文选题。
