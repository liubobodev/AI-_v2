# 工具生态总图 · 底座统一,模块配装

> 设计原则(三条,违反任何一条的工具不进体系):
> 1. **极简主义**:每个工具必须有不可替代的职责;职责重叠就淘汰一个。学生的时间花在做项目,不是学工具。
> 2. **资产回流**:任何工具的产出,最终必须回流到两个家——**知识进 Obsidian(markdown),代码进 GitHub(repo)**。回不了家的工具产出 = 不存在。
> 3. **模块配装**:底座人人必用;方向工具按学生兴趣赛道选 1 个模块,禁止贪多。

## 一、底座层(人人必用,已建成)

| 工具 | 不可替代职责 | 对应文档 |
|---|---|---|
| 教练智能体 | 调度中枢:诊断/督战/模拟面试/热点过滤 | 01_智能体设定 |
| GitHub + gh | 活水源 + 训练场 + 作品集 | 09_GitHub资源引擎 |
| Obsidian | 知识资产的家:任务卡/看板/卡点库/求职档案 | 06 + 07 |
| NotebookLM | 多模态学习引擎:播客/视频概览/测验/Debate 陪练 | 06 |
| Claude Code 或 Cursor | AI 编程主力(本身就是 JD 要求) | 第 2 关任务卡 |

## 二、方向模块层(第 1 关末选定,一人一模块)

每模块 = 3–4 个工具封顶。学生按自己在《场景选题矩阵》选的场景对号入座:

### 模块 A · 应用全栈(想做"能给人用的产品")
适配场景:1 会议纪要、4 客服助手、10 课程助教
- **Gradio/Streamlit**:Python 三行出界面,demo первый选择
- **v0 / Bolt**:自然语言生成前端,学"AI 时代的产品原型速度"
- **Vercel / Cloudflare Pages**:免费部署,让作品有真实 URL(面试官能点开 ≫ 截图)

### 模块 B · 知识库与 RAG(想进"企业知识管理/客服"赛道)
适配场景:4 客服、7 合同初审、11 内部知识库
- **Dify 或 RAGFlow**(选一,均国产开源可私有部署):低代码平台先跑通业务,再用代码复刻核心链路——"平台版 vs 手写版"对比本身就是面试题
- **Chroma**(轻量向量库,本地零成本)
- **Langfuse**:调用链观测,RAG 调优的眼睛

### 模块 C · Agent 与办公自动化(想做"数字员工")
适配场景:5 竞品监控、9 询价邮件、12 经营数据问答
- **n8n 或 扣子(Coze)**(选一):可视化工作流,理解"编排"概念最快的路
- **MCP 生态**(官方 servers + 自建):第 5 关主战场
- **飞书/钉钉机器人 API**:让 Agent 出现在真实办公软件里——"部署进企业 IM"是国内面试的强信号
- **Playwright**:浏览器自动化,采集与操作网页的手

### 模块 D · 内容与多模态(想进"新媒体/营销/设计"赛道)
适配场景:6 内容流水线
- **NotebookLM 视频概览 / Gamma**:文档→演示视频/幻灯,分钟级
- **ComfyUI 或 即梦**:图像工作流(ComfyUI 偏工程加分,即梦上手快)
- **剪映(数字人+AI剪辑)**:成片落地
- 本模块学生的 Evals 关重点做"风格 rubric + LLM judge"(无标准答案评测,难点⑧)

### 模块 E · 数据分析(想进"商业分析/经营中台")
适配场景:3 发票核验、8 质检报告、12 经营数据问答
- **代码解释器路线**(Claude/通义 code interpreter):自然语言→pandas→图表
- **text-to-SQL + 校验器**:难点③零容忍的主战场
- **Metabase**(开源 BI):让分析结果长在看板上,不是聊天记录里

### 模块 F · 评测与质量(硕士优先/想走"AI QA/评测工程"新兴岗)
适配所有场景的第 7 关深化
- **promptfoo**:评测配置即代码
- **DeepEval / Ragas**:RAG 专项指标
- **Langfuse**:线上 trace 回流评测集
- 本模块学生承担全班"评测服务台"角色——给同学的项目做第三方评测,这个经历写简历极强

## 三、连动协议(工具怎么"巧妙配合")

```
                    ┌────────── 教练智能体(调度中枢)──────────┐
                    │  口令驱动:雷达/脉搏/选题/造数据/抽问/面试  │
                    └──────┬───────────────────────┬──────────┘
      每周情报流            │                       │            学生作战流
GitHub雷达+脉搏 ─→ Obsidian日记 ─→ NotebookLM播客 │ 方向模块工具 ─→ 产出
                    ▲                              │     │
   教学生长流        │                              │     ├─ 代码/配置 → GitHub repo(四件套)
学生卡点 ─→ 卡点卡片(Obsidian)─→ Bases高频榜 ─→ 下周教学 + NotebookLM测验
                    │                              │     └─ 心得/复盘 → Obsidian作战室
                    └── 学期末:卡点库+雷达存档 = 下一届的升级教材(智能体系统提示词随之校准)
```

三条流的含义:
- **情报流**(教师主导):保证"不落伍"——见《活水来源设计》;
- **作战流**(学生主导):模块工具的一切产出回流 GitHub + Obsidian,保证"有积累";
- **生长流**(系统自动):卡点飞轮让教学内容自我进化——见《学习难点飞轮》。

## 四、教师控制面板(避免生态失控)

- 每模块指定 1 名"模块长"(学生),负责该模块工具的踩坑文档,存入 Obsidian `模块手册/`;
- 工具淘汰机制:连续 3 周没人用的工具,从模块清单移除(Bases 里给工具建"使用记录"属性即可统计);
- 新工具准入:走《活水来源设计》双源规则 + 教师试用 30 分钟能跑通,否则不进模块。
